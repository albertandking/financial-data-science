| 数据源 | 覆盖 | 门槛 | 适用 |

## 3.3 金融数据清洗的关键问题

真实行情数据"脏"在哪里？金融领域有几个**特有**的坑：

### 3.3.1 复权

公司分红、送股、配股会让价格"跳空"，但这并非真实涨跌。**复权**用于消除这种跳变：

- **前复权（qfq）**：以最新价为基准向前调整，常用于看图、算收益。
- **后复权（hfq）**：以上市首日为基准向后调整，常用于长期回测。

!!! warning "务必用复权价算收益"
    用未复权价计算收益率，会在除权日产生巨大的虚假"暴跌"。本书统一用**前复权**。

### 3.3.2 停牌与缺失

停牌日没有成交，数据缺失。处理方式取决于场景：

- 计算单只股票收益：停牌日**剔除**或前向填充价格（收益记 0）。
- 多只股票对齐：需统一交易日历，缺失处**不要**用 0 价格，应对齐后前向填充或标记。

### 3.3.3 异常值

录入错误、极端行情会产生离群点。常用识别方法：

- **涨跌停约束**：A 股主板日涨跌幅一般 ±10%（科创板/创业板 ±20%），超出即可疑。
- **统计法**：超过均值 ±n 倍标准差，或用稳健的中位数绝对偏差（MAD）。

处理时**谨慎**：金融的极端值往往是真实信息（如黑天鹅），不要轻易删除，更不要"温莎化"
掉真实风险。

## 3.4 对齐多只标的的交易日历

不同股票上市时间、停牌不同，直接拼接会错位。标准做法：

```python
import pandas as pd

# 以所有标的的交易日并集为统一日历，再前向填充价格
panel = pd.concat([s1, s2, s3], axis=1)   # 自动按日期索引对齐
panel = panel.sort_index()
panel = panel.ffill()                      # 停牌日用上一交易日价格填充
```

## 3.5 联网抓取真实数据（可选）

安装数据组后即可抓取 A 股日线：

```bash
uv sync --extra data
```

```python
import akshare as ak

df = ak.stock_zh_a_hist(
    symbol="600519",        # 贵州茅台
    period="daily",
    start_date="20230101",
    end_date="20251231",
    adjust="qfq",            # 前复权
)
```

也可以直接用本书脚本一键抓取并落盘到 `data/raw/`：

```bash
uv run python scripts/fetch_data.py --symbol 600519 --start 20230101 --end 20251231
```

!!! note "接口会变"
    第三方接口字段、限频可能随时调整。抓取失败时先重试、再考虑更新
    （`uv lock --upgrade-package akshare`）。教学演示请优先使用内置数据。
